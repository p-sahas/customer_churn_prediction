02 — System Design

Core Components

Each subsystem is independently containerised and communicates through well-defined interfaces, enabling horizontal scaling and fault isolation.

Orchestration

Apache Airflow

Scheduled DAG-based pipeline execution using DockerOperator for environment isolation. Manages dependencies, retries, and alerting across all pipeline stages.

DAGs DockerOperator Scheduling
Streaming

Confluent Kafka

High-throughput real-time event streaming for customer transaction data ingestion. Enables low-latency signal detection at scale before the processing stage.

03 — Developer Experience

Makefile-Driven Workflow

The Makefile interface abstracts infrastructure complexity. Run the full production stack — Kafka, Airflow, MLflow, training, and inference — with single commands defined in the project Makefile.

~/customer_churn_prediction
$ make automation-up # Starts Airflow + Docker pipeline services   $ make docker-build # Builds all project Docker images   $ make docker-run-all # Runs data → train → inference pipelines   $ make s3-upload-data # Uploads raw and processed data to S3   $
